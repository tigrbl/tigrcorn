# Docs index

This is the mutable documentation entrypoint.

Read in this order:

1. `../README.md`
2. `../.codex/AGENTS.md`
3. `review/conformance/state/CURRENT_REPOSITORY_STATE.md`
4. `review/conformance/CERTIFICATION_BOUNDARY.md`
5. `review/conformance/BOUNDARY_NON_GOALS.md`

Folder map:

- `ops/` — operator-facing CLI and public/programmatic surface docs
- `gov/` — governance, mutability, authoring, and release workflow
- `protocols/` — protocol-specific design and usage notes
- `comp/` — comparison matrices and companion sources
- `architecture/` — architecture descriptions
- `adr/` — architecture decisions
- `notes/` — mutable work/notes
- `review/` — conformance, state, release, and review artifacts

When in doubt, start with the current-state docs under `docs/review/conformance/state/` and then follow the relevant folder README.
