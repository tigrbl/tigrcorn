# Stricter all-surfaces-independent profile status

The authoritative package-wide certification target remains `docs/review/conformance/CERTIFICATION_BOUNDARY.md`.

This document records a stricter **non-authoritative** planning profile for surfaces that the canonical boundary intentionally keeps at `local_conformance`.

## What this stricter profile changes

The stricter profile is encoded in `docs/review/conformance/certification_boundary.all_surfaces_independent.json`.

It raises the following RFCs from `local_conformance` to `independent_certification` for planning purposes only:

- RFC 7692
- RFC 9110 §9.3.6
- RFC 9110 §6.5
- RFC 9110 §8
- RFC 6960

## Provisional bundle added in this update

This update adds a second offline remediation bundle:

- `docs/review/conformance/releases/0.3.6/release-0.3.6/tigrcorn-provisional-all-surfaces-gap-bundle/`

That bundle is generated by `tools/create_provisional_all_surfaces_gap_bundle.py`.

It maps the stricter profile's planned independent scenarios to the nearest local-conformance vectors and preserves a concrete directory layout for:

- permessage-deflate across the HTTP/1.1, HTTP/2, and HTTP/3 WebSocket carriers
- CONNECT relay coverage across the HTTP/1.1, HTTP/2, and HTTP/3 carriers
- trailer-field coverage across the HTTP/1.1, HTTP/2, and HTTP/3 carriers
- content-coding coverage across the HTTP/1.1, HTTP/2, and HTTP/3 carriers
- OCSP validation coverage for the package-owned TLS stack

Each provisional scenario is explicitly marked with:

- `provisional_non_certifying_substitution: true`
- `release_gate_eligible: false`
- `strict_profile_only: true`
- `source_local_conformance_vector`

## Current honest status

This stricter profile is **not** satisfied by the updated archive.

The canonical release gates are now green under the authoritative boundary.
Under this stricter all-surfaces-independent profile, the repository would still need thirteen additional third-party preserved artifacts for RFC 7692, RFC 9110 CONNECT / trailers / content coding, and RFC 6960.

The new provisional all-surfaces gap bundle improves repository transparency and planning, but it is not independent-certification evidence and does not change the canonical release-gate result.
The archive now also ships a separate provisional flow-control review bundle and a bundled intermediary/proxy seed corpus for the secondary repository-level partials discussed in `docs/review/conformance/SECONDARY_PARTIALS_STATUS.md`.

## Closure procedure for the stricter profile

1. Keep the now-complete canonical HTTP/3 / RFC 9220 closure work from `docs/review/conformance/INDEPENDENT_HTTP3_CERTIFICATION_STATE.md`.
2. Preserve third-party WebSocket permessage-deflate artifacts on the HTTP/1.1, HTTP/2, and HTTP/3 carriers.
3. Preserve third-party CONNECT relay artifacts on the HTTP/1.1, HTTP/2, and HTTP/3 carriers.
4. Preserve third-party trailer-field artifacts on the HTTP/1.1, HTTP/2, and HTTP/3 carriers.
5. Preserve third-party content-coding artifacts on the HTTP/1.1, HTTP/2, and HTTP/3 carriers.
6. Preserve independent OCSP policy artifacts for the package-owned TLS stack.

The machine-readable snapshot for this stricter planning profile lives in `docs/review/conformance/all_surfaces_independent_state.json`. The secondary repository-level partial snapshot lives in `docs/review/conformance/secondary_partials_state.json`.


## Phase 3 code/public-surface closure

The public strict-surface controls are no longer the main blocker. This checkpoint lands:

- `--websocket-compression`
- `--connect-policy` / `--connect-allow`
- `--trailer-policy`
- `--content-coding-policy` / `--content-codings`
- `--ssl-ocsp-mode`, `--ssl-ocsp-cache-size`, `--ssl-ocsp-max-age`, `--ssl-crl-mode`, `--ssl-revocation-fetch`
- `--ssl-alpn`

Those controls are now wired into the runtime across the applicable HTTP/1.1, HTTP/2, HTTP/3, WebSocket, and TLS / QUIC-TLS paths.

The remaining honest blocker for this stricter profile is preserved independent evidence, not the absence of the public policy surface.


## Phase 5 note

Phase 5 promotes the broader interop / flow-control evidence roots, but it does **not** by itself close the strict all-surfaces-independent overlay.
The strict overlay still lacks preserved third-party independent artifacts for the 13 scenario ids enumerated in `all_surfaces_independent_state.json`.
The new minimum certified flow-control bundle and the new minimum certified intermediary/proxy-adjacent corpus should therefore be read as **repository-strengthening evidence roots**, not as proof that the strict overlay is fully green.
